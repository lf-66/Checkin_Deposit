# 日常打卡小工具 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
日常打卡小工具

### 1.2 产品定位
一款面向个人用户的习惯养成与目标管理工具，通过游戏化的打卡机制和奖励系统，帮助用户建立良好习惯、管理零花钱、实现个人目标。

### 1.3 目标用户
- 学生群体
- 希望养成良好习惯的年轻人
- 需要管理零花钱的用户
- 追求自律和目标管理的个人

---

## 2. 核心功能模块

### 2.1 每日打卡系统

#### 功能描述
用户可以创建多个打卡计划，每日完成计划后打卡获取奖励。

#### 核心特性
- **多计划管理**: 支持创建多个打卡计划
- **打卡奖励**: 每个计划设置每日钻石奖励
- **连续打卡**: 追踪连续打卡天数（Streak）
- **补卡机制**: 支持补卡，需消耗钻石
- **时间限制**: 打卡时间限制在 06:00-22:00

#### 奖励机制
| 连续天数 | 额外奖励 |
|---------|---------|
| 3天     | 10钻石  |
| 7天     | 20钻石  |
| 15天    | 50钻石  |
| 30天    | 100钻石 |

---

### 2.2 目标管理系统

#### 功能描述
用户可以设定目标，并为每个目标设置所需金额。

#### 核心特性
- **目标创建**: 添加目标名称和价格
- **进度追踪**: 实时显示目标完成进度
- **多目标管理**: 支持同时管理多个目标

---

### 2.3 钻石与兑换系统

#### 功能描述
通过打卡获取钻石，钻石可兑换成现金。

#### 核心特性
- **钻石获取**: 每日打卡、连续打卡奖励、额外奖励
- **钻石兑换**: 按汇率将钻石兑换为现金
- **兑换历史**: 记录所有兑换操作

#### 汇率设置
- 默认汇率: 10钻石 = 1元
- 支持自定义汇率设置

---

### 2.4 储蓄罐功能

#### 功能描述
管理零花钱的存入、取出和目标储蓄。

#### 核心特性
- **存入**: 将现金存入储蓄罐
- **取出**: 从储蓄罐中取出现金（需要理由）
- **目标储蓄**: 为特定目标进行储蓄
- **交易记录**: 记录所有存取操作
- **数据统计**: 显示总资产和储蓄进度

---

### 2.5 历史记录

#### 功能描述
查看打卡历史和统计信息。

#### 核心特性
- **日历视图**: 以日历形式展示打卡记录
- **打卡详情**: 查看每日打卡计划完成情况
- **统计信息**: 显示总打卡天数、完成率等

---

### 2.6 数据管理

#### 功能描述
支持数据的导入、导出和备份。

#### 核心特性
- **自动加载**: 启动时自动加载最新的 JSON 数据文件
- **手动导入**: 支持手动导入 JSON 数据文件
- **数据导出**: 将数据导出为 JSON 文件（日期命名）
- **本地存储**: 使用 IndexedDB 和 localStorage 本地缓存

## 4. 数据模型

### 4.1 数据结构

```javascript
{
  // 目标列表
  goals: [
    {
      id: string,           // 唯一标识
      name: string,         // 目标名称
      createdAt: string,    // 创建时间
      price: number         // 目标价格
    }
  ],
  
  // 打卡计划列表
  plans: [
    {
      id: string,           // 唯一标识
      name: string,         // 计划名称
      content: string,      // 计划内容描述
      dailyDiamond: number, // 每日打卡钻石奖励
      streakRewards: {      // 连续打卡奖励
        3: number,          // 3天奖励
        7: number,          // 7天奖励
        15: number,         // 15天奖励
        30: number          // 30天奖励
      }
    }
  ],
  
  // 打卡记录
  checkins: {
    "YYYY-MM-DD": {       // 日期作为key
      planId: boolean      // 计划ID和是否完成
    }
  },
  
  // 连续打卡天数
  streaks: {
    planId: number         // 各计划的连续打卡天数
  },
  
  // 资产
  diamonds: number,        // 钻石数量
  money: number,           // 现金金额
  
  // 兑换历史
  exchangeHistory: [
    {
      id: string,
      amount: number,       // 兑换钻石数
      money: number,        // 获得现金
      rate: number,         // 汇率
      timestamp: string     // 时间戳
    }
  ],
  
  // 设置
  settings: {
    checkinCost: number,    // 补卡消耗钻石数
    lastSync: string,       // 最后同步时间
    pendingSync: number     // 待同步数量
  },
  
  // 额外奖励记录
  extraRewards: [
    {
      id: string,
      description: string,  // 奖励描述
      diamonds: number,     // 奖励钻石数
      timestamp: string     // 时间戳
    }
  ],
  
  // 历史记录
  historyRecords: [
    {
      id: string,
      date: string,         // 日期
      type: string,         // 记录类型
      description: string,  // 描述
      diamonds: number,     // 钻石变化
      timestamp: string     // 时间戳
    }
  ],
  
  // 储蓄交易记录
  transactions: [
    {
      id: string,
      type: 'deposit' | 'withdraw',  // 存入/取出
      amount: number,       // 金额
      reason: string,       // 理由
      goalId: string,       // 关联目标ID
      timestamp: string     // 时间戳
    }
  ]
}
```

## 5. 技术架构

### 5.1 技术栈
- **前端**: HTML5, CSS3, JavaScript (原生)
- **存储**: 
  - IndexedDB (主要数据存储)
  - localStorage (缓存和配置)
  - File System Access API (文件导入导出)
- **UI**: 响应式设计，适配移动端和桌面端

### 5.2 浏览器支持
- Chrome/Edge (推荐)
- 支持 File System Access API 的现代浏览器

## 7. 非功能需求

### 7.1 性能要求
- 页面加载时间 < 2秒
- 打卡操作响应时间 < 100ms
- 支持至少 1000 条历史记录流畅浏览

### 7.2 可用性要求
- 界面简洁直观
- 操作反馈明确
- 支持键盘和触摸操作
- 离线可用（数据操作后同步）

### 7.3 安全要求
- 数据存储在本地，不上传服务器
- 文件导入导出需用户授权
